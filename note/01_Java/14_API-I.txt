1. JDK 라이브러리(자바 표준 라이브러리)에 대하여
  - 지금까지 무심코 사용해 왔던 String 클래스와 System 클래스도 모두 이 이 라이브러리에 속한 클래스입니다. 
    이 라이브러리는 JDK 안에 포함되어 있는데 자바 가상기계가 자바 프로그램을 실행할 때 이 라이브러리의 클래스와 인터페이스들을 자동으로 읽어 들입니다.
2. JAVA 필수 API (Application Programming Interface)
  - JAVA는 개발자들이 편리하게 이용할 수 있는 풍부한 클래스들이 많습니다. API란, 누군가가 만들어 놓은 기능들이라고 생각하면 됩니다.
3. 문자열에 관련된 대표적 클래스 : String
 3-1. String의 이해
  ※ String은 객체 자료형입니다. 뒤에 생성자(new)를 사용하지 않고 기초 데이터를 만들 때처럼 초기화하고 있습니다.
   - 두 경우 모두 사용할 수 있습니다. (차이를 알아보겠습니다.)
     ① String string = “HelloWorld”;
     ② String string = new String(“HelloWorld~~”);
     : new로 생성하지 않고 ①번을 사용한 문자열 상수는 자동으로 힙영역에 String 객체가 생성됩니다.
     : 이미 존재하는 문자열 상수를 다시 사용한다면 이미 생성된 문자열 상수를 공유합니다.
 3-2. String의 문제점
  - 메모리를 과소비하기 때문에 프로그램의 성능도 떨어집니다.
3-3. StringBuffer와 StringBuilder의 등장
  - 이 클래스들은 모두 객체 내부에 있는 버퍼(buffer, 데이터를 임시로 저장하는 메모리)에 문자열의 내용을 저장해 두고 그 안에서 추가, 수정, 삭제 작업을 합니다.
    그렇기 때문에 이 클래스들은 이용하면 새로운 객체를 만들지 않고도 문자열 조작을 할 수 있습니다.
